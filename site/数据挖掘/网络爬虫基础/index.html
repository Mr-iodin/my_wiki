    网络爬虫基础

    数据获取与预处理

    爬虫

    数据获取途径途径：网络爬虫 网络服务提供的API接口。

    互联网数据：电商数据、网络新闻数据、社交数据。

    网络爬虫：

    ​ 网络爬虫通常从一个称为种子集的URL集合开始运行，它首先将这些URL全部放到一个有序代爬序列里，按照一定顺序从中取出URL并下载所指向的页面。分析页面内容，提取新的URL并放入待爬的URL队列中。重复以上过程，直到URL队列为空或满足爬行的终止条件，从而遍历web。

    爬虫技术通用框架：

    模块 功能
    URL Frontier 爬虫系统的调度器，也是其他模块的驱动。
    DNS Resolve 机型域名解析，并输入URL Frontier，一个循环完成后，URL Frontier返回网页内容。
    RIS 远程服务器，实现爬虫与互联网Internet的通信。
    Content Seen 对网页内容进行去重。
    Extractor 获取当前页面的URL，并提取。
    URL Filter 一般用于垂直应用，将Extractor输出的URL进行筛选，过滤无效的URL。
    Tag Counter 标签计数。
    URL Seen 对URL去重。
    URL Set 一般是硬盘、数据库等用于存取URL。

    HTML

    用于描述网页文档的一种标记语言。

    网页文档的结构和格式的定义是由HTML元素来完成的，HTML元素是由单个或一对标签定义的包含范围。

    HTML DOM

    文档对象模型。定义了访问和处理HTML文档的标准方法。

    节点间关系： 除文档结点外的每个节点都有父节点，大部分元素都有子节点。

    同辈： 分享同一个父节点的节点是同辈。

    正则表达式

    • 测试字符串内的模式。

    可以测试输入的字符串，以查看字符串内是否出现电话号码模式或信用卡模式。这称为数据验证。

    • 替换文本。

    可以使用正则表达式识别文档中的特定文本，完全删除该文本或者用其他文本进行替换。

    • 基于模式匹配从字符串中提取字符串。

    可以查找文档内或输入域特定文本。

    例如，在搜索整个网站需要删除某些文本，以及替换某些HTML格式标记。在这种情况下，可以使用正则表达式来确定在每个文本中是否出现该文本或HTML标记。然后可以使用正则表达式来删除这些文本或格式标记。当然，可以使用正则表达式来搜索和替换标记。

    requests:

    1.伪装自己所需要的参数：

    Headers={"User-Agent":用户代理头，浏览器自己的标识符,
         "Cookie":有时抓取需要登录的网站，需要设置cookie
        }
param={"param1":get请求传递参数}
Data={"data":post请求传递的数据}

    2.发送HTTP请求

    GET请求：req=request.get(url,headers,param)
POST请求：req=request.post(url,headers,data)

    3.获取响应

    如果返回的是文本文件（html）,用req.text
如果返回的是json文件数据，用req.json

    Selenium模拟器：

    Selenium是一种自动化的测试工具，可以用它来完成几乎所有的浏览器操作，比如点击、输入等。它支持各种浏览器，包括Chrome、Safari、Firefox等主流界面式浏览器。

    有时候网页中包含着一些javascript动态加载元素，这些内容是用户在浏览器打开网页的时候才会执行的，所以如果直接用requests去请求，是无法完成JS的这些动作的，也就无法获得我们想要的内容，所以必须模拟浏览器的作用。

    browser=webdrive.Chrome() #这一步操作会打开一个chrome浏览器
browser.get("https://www.iodin.cn/") #这一步会在打开的浏览器请求该博客

    Scrapy :

    Scrapy是一个为了爬取网页数据，提取结构性数据而编写的应用框架。

    使用Scrapy这种成熟框架的好处就是，很多重复而又重要的工作框架里都有成熟的解决机制，不需要我们去思考。

    • cookies and session
    • user-agent模拟
    • robots.txt处理
    • 爬取深度限制
    • 爬取速度限定
    • 提供了自动检测以及健壮的编码支持，所以可以放心抓中文网页
    • 提供了media.pipeline，可以自动下载 爬取到数据中的图片（或者其他资源）

    API基础与原理

    API,用户程序编程接口。是一种预先定义的函数，目的是提供应用程序与开发人员基于某软件或硬件得以访问一组例程的能力，而又无需访问源码，或理解内部工作机制的细节。

    为不同应用提供友好的接口。不同的开发者用不同的架构，甚至不同的语言编写软件都可以通过API实现共享。因为API设计的目的就是成为一种通用语言，让不同软件进行信息共享。

    ​ 一般情况下，程序员可以用HTTP协议向API发起请求以获取某种信息，API会用XML（可扩展标记语言）JSON(javascript对象表示) 格式返回服务器响应信息。

    ​ 和大多数网络数据采集方式不同，API用一套非常标准的规则生成数据，而且生成的数据也非常标准的方式组织的。

    ​ 因为规则很标准，所以简单、基本规则很容易学，可以快速地掌握任意API的用法。

    方法：GET、POST、PUT、DELETE

    • GET：用于获取URL资源（统一资源标识符：Uniform Resource Identifier）的进行展示，GET操作不会对服务器端有任何影响。
    • POST：用于新建资源，服务端在新的URL上创建一个新的对象，将新资源的地址作为响应信息的一部分返回；每次当你登录网站的时候，就是通过用户名和密码发起一个POST请求，相当与把信息保存到数据库中。
    • PUT：用于更新URI上的一个资源，如果服务器端允许，PUT也可以新建一个资源；例如，API可能会要求用POST请求创建新用户，但是如果要跟新老用户邮箱地址，就要用到PUT请求了。
    • DELETE：DELETE用于删除指定的URI资源。DELETE方法在公共API里边不常用，不能随便让一个用户去删除掉数据库的信息。

    API验证——token:

    无验证：即任何人都需要使用API，不需要注册。

    有验证：

    • 有些API要求客户验证是为了计算API调用的费用，或者是提供包月服务。
    • 有些验证是为了”限制“用户使用API（限制每秒钟、每小时或每天API的调用次数），或者限制一部分用户对某种信息或某种信息或某类API的访问。

    通常API验证方法都是用类似token的方法调用，每次API调用都会把token传递到服务器上。这种token要么注销时分配给用户，要么是在用户调用时才会提供，提供的口令是固定的，也可能是频繁变化。通过服务器对用户名和口令密码的组合生成。

    API有一个重要特征是它们会反馈格式友好的数据。大多数反馈格式都是XML和JSON。

    • XML：可扩展标记语言，标准统用标记语言子集，是一种用于标记电子文件使其具有结构性的标记语言。它可以用来标记数据、定义数据类型，是一种运行用户对自己的标记语言进行定义的源语言。它非常适合万维网传输，提供统一的方式描述和交换独立应用程序或供应商的结构化数据。
    • XML格式统一，符合标准；
    • 容易和其他系统进行远程交互，数据共享比较方便；
    • XML文件庞大，文件格式复杂，传输占宽带；
    • 服务器端和客户端都需要花大量代码解析XML；
    • JSON：JS对象标记，是一种轻量级的数据交换格式。它基于ECMAS（w3c制定的js规范）的一个子集，采用完全独立于编程语言的文本格式来存储和表示数据。
    • JSON数据结构比较简单，易于读写，格式都是压缩的，占用宽带小；
    • JSON易于解析，客户端JavaScript以简单的通过eval()进行JSON数据读取；
    • 支持多种语言，便于服务器端的解析；
    • JSON应用推广在不断进步；
