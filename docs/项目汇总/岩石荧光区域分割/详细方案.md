# 二、详细方案

## Unet图像分割——岩石荧光区域

## 摘要

**针对第二问：**对岩石含油荧光区域精准分割。本方案首先对原始数据提供的石油在紫外线照射下具有发光特征图像，根据分析"绿色/黄色"特征在四种**色彩空间**下不同通道的直方图分布差异，选择在LAB和YCrCb空间下采用阈值分割获得初步分割结果。然后，采用基于全卷积神经网络的**U-net框**架，通过增强图像色度、图像增广等手段提高模型特征学习的拟合度，识别目标区域掩膜，并采用**dice系数**对模型识别结果进行评估，（评估指标）。最后，结合**GrabCUt算法**对部分图像进行前后景分割，最终计算得到岩石的含油面积百分含量。

## 二、基本理论

### 2.2 问题二

**色彩空间**

​	颜色是物体表面对固有特征，在处理颜色信息时，机器视觉系统需要依据关于物体表面光谱反射性质的稳定描述， 这就涉及到颜色恒常性 、颜色空间和颜色测量等基础问题<sup>[2_1]</sup>。

​	色彩空间，又称为颜色坐标系，是为计算机视觉系统建立的一套色彩模型。基本结构可以分两大类：基色颜色空间 和色、亮分离颜色空间。本方案涉及到的主要有以下四种：

* RGB：是面向硬件设备中最常用的颜色空间，由3个固定分量——R(红)、G（绿）、B（蓝）组成。3种基本色的波长分别为700nm、564.1nm和435.8nm。
* LAB：是由国际照明委员会所制定的具有国际标准的色彩模式，由一个亮度通道L和两个颜色通道A和B组成，其中，L代表亮度，A代表颜色从绿色到红色的转变，B代表从蓝色到黄色的转变。
* YCrCb：主要应用于电子类视频信号的传输。由一个亮度通道Y和两个颜色通道Cr和Cb组成。其中Y表示伽马矫正后从RGB获得的亮度分量，Cr表示与Luma的红色分量距离，Cb表示与Luma的蓝色分量距离。
* HSV：一种均匀的颜色空间, 反映了人的视觉对颜色的感知。其中L表示亮度信息，H表示色调信息，S代表饱和度信息。

**卷积神经网络**

​	卷积神经网络(Convolutional neural networks, CNN) 是一种带有卷积结构的深度神经网络，其特有的卷积结构能在一定程度上减少深层网络占用的内存量与网络参数个数，缓解模型的过拟合问题。<sup>[2_2]</sup> 基本框架由主要由输入层，卷积层，池化层（也称为采样层），全连接层及输出层构成，如图（2.2.1.1）为一个简单的CNN框架：

<img src="https://i.loli.net/2021/03/21/ILC6cPUz7kKZbFq.png" style="zoom: 33%;" />

<center>【图（2.2.1.1）由2个卷积层、2个子采样层交替组成CNN框架】</center>

* 数据输入层（Input Layer）：主要对传入数据进行一些简单的预处理。

* 卷积层（Convolution）：经过输入层经过简单的规则化后，该层数据就可以被认为是一个可学习的卷积核进行卷积, 卷积层会将该层的单元的一组的局部近邻的单元作为输入，通过一个激活函数(Activation function),构建出一幅特征图(Feature map) 。
  
  ![](https://s2.loli.net/2022/03/10/npu2K3QfaHXegRx.png)
  
  * 上标l表示第l层；
  * $x^{l-1}_{i}$为上层特征图像的某个像素值；
  * $u^{l}_{j}$称为卷积l的第j个通道的净激活 (Netactivation), 由对前一层输出的特征图$x^{l-1}_{i}$进行卷积求和偏置后($b^l_j$为偏置项)得到；
  * $x^l_j$是卷积层l的第j个通道输出的;
  * $M_j$表示参与运算的上一层特征图像子集；
  
* 池化层（pooling）：池化层夹在连续的卷积层中间， 用于压缩数据和参数的量，减小过拟合。卷积层后一般会接入池化层来减小数据量，通过交替的卷积层和池化层的处理，来获得高度抽象的特征图像。

* 全连接层（fully connected layers）：在卷积神经网络中其到分类器的作用，作用为将学到的“分布式特征表示”映射到样本标记空间。

* 输出层（Output Layer）：负责对结果进行输出。

**全卷积神经网络**

​	全卷积神经网络(Fully Convolutional Network，FCN)是用于图像语义分割所提出的一种框架。全卷积神经网会对图像进行像素级的分类，与CNN中全连接层得到的固定长度分类（全联接层＋softmax输出）不同，FCN是一个输入级端对端的学习结构<sup>[2_3]</sup>，可以接受任意尺寸图像，并采用反卷积对最后一个卷积层feature map进行上采样，使全连接层转化为卷积层，获得与原数据空间一致的特征，保留其图像的空间结构。

## 五、岩石含油荧光区域分割统计

### 5.1 基于U-net荧光分割模型

​	在传统的阈值通道分割法颜色检测系统中，阈值的选择都是需要对大量原始数据图像的统计分析而得到，人工分割阈值往往具有很大的片面性和主观性，这导致了阈值的选取范围波动较大，不具有普适性；同时，人工划分阈值的方法往往需要花费大量人力与时间，这也导致该方法不能直接运用于需要处理大量数据的工程应用中。

​	因此在这一部分，我们将采用深度学习的方法U-Net框架并结合GrabCUt算法对岩石荧光区域进行分割，以此减少传统阈值分割法带来的人力与时间消耗，最大程度的提高分割效率。

### 5.1.1 U-net网络

​	在统计岩石含油的荧光面积之前，我们首先要做的便是确定目标区域。我们把荧光区域作为我们的目标区域，以此将图像的面积统计问题转化为图像二值分割问题，最后得到的分割图像中目标区域像素点的占比即为本方案的最终结果。在本方案中，确定目标区域的任务采用的是U-net框架来设计。

​	U-net网络是基于FCNs做的优化，相较于FCN，优势主要有以下两点：1）在解码器部分添加了反卷积操作和上采样模块，并添加了很多特征通道，允许原图像纹理信息在分辨率的卷积层中进行传播；2）采用拼接方式进行特征融合以便更好的利用编码器部分不同尺度信息。<sup>[2_4]</sup>

整体结构图如图（5.1.1.1）所示： 

![u-net](https://i.loli.net/2021/05/06/U2q67iKLBDzHr5E.png)

​	<center>【图（5.1.1.1） U-net整体结构图】</center>

Unet整个网络呈现为一个对称的U型结构，其左半部分和右半部分分别为“编码”和“解码”（Decoder）过程：

* 编码（Encoder）：由两个3×3卷积层和2×2的max polling 反复叠加组成，每经过一次下采用，通道数就会翻一倍；
* 解码（Decoder）：由一个2×2的上采样卷积层、Concatenation（由crop对应的Encode层输出feature map然后与Decode层上采样结果的叠加）和两个3×3卷积层反复叠加构成；

最后一层通过一个1×1卷积将通道数变为期望的类别数。

本方案中采用U-net网络详细结构如下：

<img src="https://i.loli.net/2021/05/05/XbGYOLkpeamcTDM.png" style="zoom:50%;" />

<center>【图（5.1.1.2） 本方案中采用U-net网络详细结构】</center>

​	首先，为了提高梯度下降速度，减少训练所需时间，我们选用Relu函数作为整个网络的激活函数，函数公式如下：
$$
f(x)=max(0,x)
$$
 Relu函数是一种非线性饱和的函数，相较与传统的神经网络激活函数，其函数特性能实现更有效率的梯度下降以及反向传播，避免梯度消失与梯度爆炸问题。

<img src="https://i.loli.net/2021/05/06/WfQ5Brjuk9LUmtD.png" alt="截屏2021-05-06 下午1.38.54" style="zoom:33%;" />

<center>【图（5.1.1.3） Relu函数图像】</center>

​	对于图（5.1.1.2），在整个网络的编码部分，我们将"256 ×256×3"分割好后的数据集图像输入网络，每经过一次池化层下采样后，channels就会变为原来的2倍。在解码部分，扩张路径由2×2反卷积构成，反卷积的输出通道为原来通道数的一半，再与原来的feature map串联拼接，得到和原来通道数一样多的feature map，然后再经过2个尺寸3×3卷积和ReLu的作用。在最后一层中，通过一层卷积核大小为1×1的卷积作用将通道变为期望的两类，即为本方案所需要的荧光区域二分类图像。



### 5.2 GrabCUt算法前后景识别

​	进一步探究第二类图像，此类图像任务不仅需要对含油荧光区域进行分割，同时还要分析岩石在整个图像中的面积占比，所以在这一部分，我们将讨论如何使用 GrabCUt算法高效将岩石主体与背景相分离，实现一个前景与主体分割任务。

**Graph cut算法**

​	图像分割的目的就是从图像背景中提取出我们想要的目标区域，在前景分割任务中，交互式分割通过图像中的纹理（颜色）信息和边界（反差）信息并结合用户交互操作来获取的分割结果。这里首先介绍一下交互性算法Graph cut在前景分割中的原理。

​	Graph cut算法是一种基于组合最优的交互式算法<sup>[2_5]</sup>。该方法将图像分割问题与图的最小割（min cut）问题关联起来，首先需要用户选择一部分像素作为目标，其余作为像素背景，得到标签结合为$L=\{0,1\}$,构建一个加权图，其中，图的结点代表原图像的像素点，用$f_p$代表图像像素$p\in V$的标记，则整个标记过程就是一个从$V$到$L$的映射，记为$f=\left\{ f_{p}|f_{p}\in L\right\}  $，最后，通过最小化Gibbs能量函数就能解决前景分割这样一个二分类问题，即：
$$
E(f)=R(f)+\lambda B(f)
$$
其中，R(f)是数据项，用来衡量所有结点的标记情况；B(f)是边界项，用来表示边界属性；$\lambda$为比例因子，用来控制区域项和边界项所占的比重，当$\lambda$很小时，数据项容易被忽略，而当$\lambda$很大时，所有结点将具有同样标记。

**GrabCUt算法分割岩石图像前景**

​	GrabCut是对Graph cut的优化，通过不断迭代Graph cut算法，深化图像特征信息，只需少量的用户交互操作便可得到比较好的分割效果。<sup>[2_5]</sup>且相较与Graph cut逐像素点繁琐的标记方法，GrabCut算法只需要用方框标记出背景，其余部分都作为待定部分，同时在边缘部分，Graph cut还可以结合matting技术，进行精细的渡边处理。<sup>[2_6]</sup>

​	整个算法过程总结如下：

1. 如图（5.2.1.1），用户首先通过一个方框来选择初始 trimap T，方框外的像素都被默认为背景像素$T_b$，方框内的像素$T_U$看作“可能”目标像素；

2. 对$T_B$内的每一像素n，初始化像素n的标签$α_n=0$，作为为背景像素；对$T_U$内的每个像素n，初始化像素n的标签$α_n=0$，作为“可能”目标像素;

3. 迭代最小化

   1).对未知区域的每个像素分配GMM中的高斯分量，前后景分别设置为K个GMM组件：
   $$
   k_{n}:=arg\  min\  D\left( \alpha_{n} ,k_{n},\theta ,z_{n}\right)
   $$
   2).通过属于该高斯分量的像素个数与总的像素个数的比值来确定GMM参数
   $$
   \theta =\  arg\  min\  U\left( \alpha ,k,\theta ,z\right)  
   $$
   3).分析Gibbs能量项，建立一个图，并求出权值t-link和n-link，通过max flow/min cut算法来进行分割:

   ![](https://s2.loli.net/2022/03/10/goRhxQqaNdJXEnK.png)

   4).重复步骤1）到3），直到收敛。

   5）应用border matting对分割的边界进行平滑等后期处理。

4. 用户编辑与重操作。

##### 6.2.2 评价指标

​	Dice系数是一种集合相似度度量函数，通常用于计算两个样本的相似度，取值范围在[0,1]。本方案中的岩石荧光分割模型将采用Disc系数进行评估，指标具体含义如下：
$$
Dice=\frac{2TP}{2TP+FP+FN} =\frac{2\left| X\cap Y\right|  }{\left| X\right|  +\left| Y\right|  }
$$
​	其中，X为第一批手工标记掩膜数据，Y为模型预测所得到的掩膜数据。

```python
smooth = 1. # 用于防止分母为0.
def dice_coef(y_true, y_pred):
    y_true_f = K.flatten(y_true) # 将 y_true 拉伸为一维.
    y_pred_f = K.flatten(y_pred)
    intersection = K.sum(y_true_f * y_pred_f)
    return (2. * intersection + smooth) / (K.sum(y_true_f * y_true_f) + K.sum(y_pred_f * y_pred_f) + smooth)

def dice_coef_loss(y_true, y_pred):
    return 1. - dice_coef(y_true, y_pred)
```

```python
def dice_coef_theoretical(y_pred, y_true):
    """Define the dice coefficient
        Args:
        y_pred: Prediction
        y_true: Ground truth Label
        Returns:
        Dice coefficient
        """
    y_true_f = tf.cast(tf.reshape(y_true, [-1]), tf.float32)
 
    y_pred_f = tf.nn.sigmoid(y_pred)
    y_pred_f = tf.cast(tf.greater(y_pred_f, 0.5), tf.float32)
    y_pred_f = tf.cast(tf.reshape(y_pred_f, [-1]), tf.float32)
 
    intersection = tf.reduce_sum(y_true_f * y_pred_f)
    union = tf.reduce_sum(y_true_f) + tf.reduce_sum(y_pred_f)
    dice = (2. * intersection) / (union + 0.00001)
 
    if (tf.reduce_sum(y_pred) == 0) and (tf.reduce_sum(y_true) == 0):
        dice = 1
 
    return dice
```
