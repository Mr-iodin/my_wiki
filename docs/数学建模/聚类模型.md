# 几种聚类模型

将样本划分为由相似对象组成的多个类的过程。

聚类后可以更加准确的在每个类中单独使用统计模型进行评估、分析和预测；探究不同类之间的相关性和主要差异。

### K-MEANS 聚类算法（K-均值聚类）

#### 流程

1.指定划分的簇的个数K值（类的个数）；

2.随机的选择K个数据对象（或空间中随意的K个数值点）作为初始的聚类中心；

3.计算其余各数据对象到这个K个初始聚类中心的距离，把数据对象划分到**距离它最近的**那个中心所处在的簇类中；

4.调整新类并且**重新计算出这个新类的中心**；

5.循环 3 和 4，看中心是否收敛，如果收敛或达到迭代次数则停止循环；

6.结束。

![K-均值聚类步骤.png](https://i.loli.net/2020/03/29/sGvZAoaf9xIenPc.png)

![K-均值聚流程.png](https://i.loli.net/2020/03/29/JMIYo8BGqLlvutR.png)

#### 优缺点

优点：

* 算法简单、快速，效率高；
* 对处理大数据集，该算法相对高效；

缺点：

* 要求用户事先必须给出生成簇的数目；
* *对初值敏感（初值选择不同，对结果有较大影响）；
* *对于孤立点数据敏感（新的聚类中心容易受到异常点影响）；

K-均值++可以解决第 2 和 3 问题。

### K-means++

它只对K-means算法“初始化K个聚类中心”这一步进行了优化。

K-均值++选择初始聚类中心的基本原则是：**初始的聚类中心之间的距离尽可能远。**

#### 流程

1.随机选取一个样本点作为中心点；

2.计算每个样本点与当前已有聚类中心的最短距离（即与最近的一个聚类中心的聚类），这个值越大，表示被选取作为聚类中心的概率越大；最后用轮盘法（依据概率大小来进行抽取）选出下一个聚类中心；

关于这个方法的可行性理解：

若要把所有的样本点划分到不同的类中，则要保证类与类之间的距离尽可能的大。而聚类中心则表示类与类之间的中心，只要保证初始的聚类中心距离足够远的话，实际上就能保证不同的类之间距离足够的大；

3.重复2，选出K个聚类中心。选出初始点后，就继续使用K-均值算法。



在spss中默认使用k-均值++。

### 系统聚类

系统聚类的合并算法通过计算两类数据点间的距离，对**最为接近的两类数据点进行组合**，并反复迭代这一过程，直到所有数据点合成一类，生成**聚类谱系图**。

因为要找出最接近的两点，所以需要计算样本点之间的距离。

#### 类与类之间的距离

一开始是用样本之间的距离，但有了类之后就不一样了。

由一个样本组成的类是最基本的类；如果每一类都由一个样本组成，那么样本之间的距离就是类之间的距离。

如果某一个类包含不止一个样品，那么就要确定类之间的距离，类之间的距离都是基于样本之间的距离定义的，大致有以下几种定义方法：

* 记号：G<sub>p</sub>和G<sub>q</sub>是两个类，D（G<sub>p</sub>，G<sub>q</sub>）是两个类之间的距离。x<sub>i</sub>∈G<sub>p</sub>  x<sub>j</sub>∈G<sub>q</sub> ,d(x<sub>i</sub>,x<sub>j</sub>)是这两个样品的距离。

#### 系统（层次）聚类流程图

![](https://pic.downk.cc/item/5e8079a7504f4bcb0445d70f.png)

1.将每个对象看作一类，计算两两之间最小距离；

2.将距离最小的两个类合并成一个类；

3.重新计算新类与所有类之间的距离；

4.重复 2 3 2 步，直到所有类最后合并成一类；

5.结束。

### DBSCAN算法

一种基于密度的聚类算法，聚类前不需要预先指定聚类个数，生成簇的个数不定（和数据有关）。

该算法利用基于密度的聚类概念，即要求聚类空间中的一定区域内包含对象（点或其他空间对象）的数目不小于某一给定阈值。

该方法能在具有噪声的空间数据库里发现任意形状的簇，可将密度足够大的相邻区域连接起来，能有效处理异常数据。

（谁离我挨得近，我就是谁兄弟，兄弟的兄弟，也是我兄弟）

#### 基本概念

DBSCAN算法将数据点分为三类：

* 核心点：在半径Eps内含有不少于MinPts（最少点数目）数目的点；
* 边界点：在半径Eps内点的数量小于MinPts，但是落在核心点的领域内；
* 噪音点：既不是核心点也不是边界点的点；

![DBSCAN算法三类点.png](https://i.loli.net/2020/04/11/8iTn7M1g6RSVGlO.png)

#### 优缺点

**优点：**

* 基于密度定义，能处理任意形状和大小的簇；
* 可在聚类的同时发现异常点；
* 与K-means比较起来，不需要输入要划分的聚类个数；

**缺点：**

* 对输入参数ε和Minpts敏感，确定参数困难；
* 由于DBSCAN算法中，变量ε和Minpts是全局唯一的，当聚类的密度不均匀时，聚类距离相差很大时，聚类的质量较差；
* 当数据量大时，计算密度单元的计算复杂度大；

**所以，使用DBSCAN时注意：**

只有两个指标，且做出散点图后发现数据呈现“块状”分布，这时再用DBSCAN进行聚类。

其他情况下尽量全部使用系统聚类。
